## Project Overview
The **Student Marks Management System** is a web-based CRUD application built using Java Server Pages (JSP) and PostgreSQL. It provides an intuitive interface for managing student marks records. Authorized users can log in, view all student entries, add new records, update existing ones, and delete records. The application demonstrates session handling, JDBC connectivity, and responsive UI design with Bootstrap.

## Technologies Used
| Technology           | Purpose                               |
|----------------------|---------------------------------------|
| Java (JSP, Servlets) | Backend logic and dynamic content     |
| HTML5, CSS3          | Structure and styling                 |

| JavaScript           | Client-side interactions and alerts   |
| PostgreSQL           | Database management                   |
| Apache Tomcat        | Web server and servlet container      |
| JDBC                 | Database connectivity                 |

Application Features
--------------------
| Feature        | Description                                                                 |
|----------------|-----------------------------------------------------------------------------|
| User Login     | Secure login with session management. Default credentials: `admin` / `admin123`. |
| View Records   | Displays all student marks in a Bootstrap-styled table.                     |
| Add Record     | Modal form to insert new student details (name, gender, subject, marks).    |
| Update Record  | Pre-filled form to edit existing student information.                       |
| Delete Record  | Deletes a record after confirmation alert.                                  |
| Logout         | Invalidates session and redirects to login page.                            |

File Structure and Description
------------------------------
| File Name       | Description                                                                 |
|-----------------|-----------------------------------------------------------------------------|
| `index.html`    | Professional login page with username/password fields. Styled with Bootstrap. |
| `login.jsp`     | Processes login credentials, queries `students_new` table, and sets session. |
| `home.jsp`      | Main dashboard: displays student table, add modal, and handles insertion.   |
| `delete.jsp`    | Deletes a record based on `id` parameter and redirects to home.             |
| `update.jsp`    | Fetches record by `id` and displays an editable form (calls `updt.jsp`).    |
| `updt.jsp`      | Processes the update from `update.jsp` and redirects to home.               |
| `nupdate.jsp`   | Alternative update page (similar to `update.jsp` but used by another flow). |
| `nupdt.jsp`     | Processes the update from `nupdate.jsp`.                                    |
| `logout.jsp`    | Invalidates session and redirects to `index.html`.                          |

> **Note:** Both `update.jsp`/`updt.jsp` and `nupdate.jsp`/`nupdt.jsp` serve the same purpose. You can consolidate them as needed.

How to Use
----------
### 1. Login
- Open the application in your browser.
- Enter username and password (default: `admin` / `admin123`).
- Click **Login**.

### 2. View All Records
- After successful login, you will see the `home.jsp` page with a table listing all students.

### 3. Add a New Record
- Click the **Add Student Marks** button.
- Fill in the details in the modal form.
- Click **Save changes**.
- A success/error alert will appear, and you will be redirected to the home page.

### 4. Update a Record
- In the table, click the **Update** button next to the record you want to edit.
- Modify the fields in the update form.
- Click **Update** to save changes.
- An alert will confirm success/failure.

### 5. Delete a Record
- Click the **Delete** button next to the record.
- Confirm the deletion in the pop-up alert.
- The record will be removed, and the page will refresh.

### 6. Logout
- Click the **Logout** button at the top-right corner of the home page.
- You will be redirected to the login page.

Troubleshooting
---------------
| Issue                                      | Solution                                                                                   |
|--------------------------------------------|--------------------------------------------------------------------------------------------|
| Database connection error                  | Verify PostgreSQL service is running. Check database name, username, and password in all JSP files. |
| `ClassNotFoundException` for Driver        | Ensure PostgreSQL JDBC JAR is placed in `WEB-INF/lib` and added to build path.             |
| Login always fails                         | Confirm that `students_new` table contains a valid user. Check SQL query syntax in `login.jsp`. |
| Insert/Update not working                  | Ensure all request parameter names match (`snm`, `gen`, `sub`, `mark`). Check for typos.   |
| 404 errors on JSP pages                    | Verify all files are in the correct `WebContent` folder and the URL is correct.            |
| Modal not opening                          | Make sure Bootstrap and Popper.js scripts are correctly linked in the JSP files.           |
| Session expired or not set                  | Check that `session.setAttribute` is called in `login.jsp` and that `logout.jsp` properly invalidates. |
